# 编译原理专题实践：接口设计与协作文档包

> 适用对象：3人小组，项目内容包括 Lex / Yacc 输入文件解析、自动机、LR/LALR、符号表、类型检查、中间代码生成。
>
> 目标：在一开始就统一接口、数据结构、目录、联调方式、测试规范和文档模板，减少后期返工。

---

# 1. 项目总目标与模块边界

## 1.1 总流程

```text
.l / .y 输入文件
   ↓
Lex输入文件解析        Yacc输入文件解析
   ↓                      ↓
正则规则/Token规则       文法/优先级/语义动作规则
   ↓                      ↓
NFA → DFA → MinDFA       First/Closure/GOTO/LR表/LALR表
   ↓                      ↓
词法分析器               语法分析器
         ↓ token流 ↓
         抽象语法树 AST / 规约序列
                 ↓
           语义分析模块
      (符号表 / 类型检查 / IR生成)
                 ↓
        符号表 / 语法树可视化 / 四元式
```

## 1.2 三人模块划分

### 成员A：Lex与自动机
- `.l` 文件解析
- 正则表达式解析
- NFA / DFA / 最小化
- 词法分析执行器
- 输出 Token 流

### 成员B：Yacc与语法分析
- `.y` 文件解析
- 文法与优先级管理
- First / Follow（如需要）
- LR(1) Item / Closure / GOTO
- LALR 状态合并
- 语法分析表与语法分析器
- 输出 AST / 规约序列

### 成员C：语义与集成
- 符号表
- 类型表达式与类型检查
- 语义动作执行框架
- 四元式 / 三地址码生成
- 系统联调、统一 CLI、测试整合

# 3. 核心接口设计

> 原则：
> 1. 模块之间只传“稳定的数据结构”，不要互相读对方内部变量。
> 2. 一切接口都要有统一返回值和错误结构。
> 3. 先定义最小可运行接口，再逐步扩展。

---

## 3.1 公共返回值接口

### Result<T>

```cpp
struct ErrorInfo {
    int code;                 // 错误码
    std::string message;      // 错误消息
    int line;                 // 源文件行号，没有则为-1
    int column;               // 列号，没有则为-1
    std::string stage;        // lex / yacc / semantic / ir
};

template<typename T>
struct Result {
    bool ok;
    T data;
    std::vector<ErrorInfo> errors;
    std::vector<std::string> warnings;
};
```

### 统一错误码建议

```text
1000-1999  Lex输入文件解析错误
2000-2999  Regex/自动机构造错误
3000-3999  Yacc输入文件解析错误
4000-4999  LR/LALR构造错误
5000-5999  源程序词法错误
6000-6999  源程序语法错误
7000-7999  符号表/作用域错误
8000-8999  类型检查错误
9000-9999  中间代码生成错误
```

## 3.2 Token 接口（A → B → C）

### TokenType

```cpp
enum class TokenType {
    IDENTIFIER,
    KEYWORD,
    INTEGER_LITERAL,
    FLOAT_LITERAL,
    CHAR_LITERAL,
    STRING_LITERAL,
    OPERATOR,
    DELIMITER,
    END_OF_FILE,
    UNKNOWN
};
```

### Token

```cpp
struct Token {
    TokenType type;           // 粗粒度类别
    std::string kind;         // 细粒度种别，如 ID / NUM / IF / PLUS
    std::string lexeme;       // 原始字符串
    int line;
    int column;
    int length;
    int symbol_id;            // 若是标识符，可关联符号表入口，未绑定则为-1
    std::string value_repr;   // 字面值字符串形式，如 "123" / "3.14"
};
```

### TokenStream

```cpp
struct TokenStream {
    std::vector<Token> tokens;
};
```

### A 模块对外接口

```cpp
Result<LexSpec> parseLexSpec(const std::string& lexFilePath);
Result<RegexAST> parseRegex(const std::string& regexText);
Result<NFA> buildNFA(const RegexAST& regexAst, const std::string& tokenKind);
Result<DFA> determinize(const NFA& nfa);
Result<DFA> minimizeDFA(const DFA& dfa);
Result<TokenStream> tokenizeSource(const DFA& dfa, const std::string& sourceCode);
```

### A 输出给 B 的唯一标准

- B **只认 TokenStream**
- B **不直接调用 A 的内部 NFA / DFA 结构**
- 词法优先级规则：
  1. 最长匹配优先
  2. 匹配长度相同时，规则文件中靠前者优先

### ParseResult

```cpp
struct ParseStep {
    int stepNo;
    std::vector<int> stateStack;
    std::vector<std::string> symbolStack;
    Token lookahead;
    std::string action;      // shift s5 / reduce p3 / accept / error
};

struct ParseResult {
    bool accepted;
    std::vector<ParseStep> steps;
    std::vector<int> reducedProductionIds;
    struct ASTNode* astRoot;
};
```

### B 输出给 C 的唯一标准

- C **主用 `ASTNode* astRoot`**
- C **辅用 `reducedProductionIds`** 用于调试
- 语义动作最终以 AST 上附加属性的方式执行，不依赖 B 的栈内部细节

```cpp
enum class TypeKind {
```

### 作用域


### 符号表管理接口

```cpp
class SymbolTableManager {
public:
    int enterScope(ScopeKind kind, const std::string& name);
    void exitScope();
    Result<SymbolEntry*> insert(const SymbolEntry& entry);
    SymbolEntry* lookupCurrent(const std::string& name);
    SymbolEntry* lookupVisible(const std::string& name);
    Scope* currentScope();
    std::vector<Scope> dumpAllScopes() const;

```

### 统一规则

1. 进入 `{ ... }` block 必新建作用域。
2. 函数参数单独进入函数作用域。
3. 变量 offset 按声明顺序递增。
4. 同一作用域内重定义报错；内层遮蔽外层允许。

## 3.6 类型检查接口（C内部）

```cpp
Result<std::shared_ptr<TypeExpr>> synthesizeExprType(ASTNode* exprNode, SymbolTableManager& stm);
Result<void> checkAssignment(ASTNode* lhs, ASTNode* rhs, SymbolTableManager& stm);
Result<void> checkFunctionCall(ASTNode* callNode, SymbolTableManager& stm);
Result<void> checkStatement(ASTNode* stmtNode, SymbolTableManager& stm);
Result<void> checkProgram(ASTNode* root, SymbolTableManager& stm);
```

### 最小支持规则

1. 算术表达式：`int + int -> int`，`int + float -> float`
2. 赋值：左右类型必须一致或允许安全拓宽
3. 条件表达式：结果必须可判真值
4. 数组下标：下标必须为整数
5. 函数调用：参数个数匹配、类型兼容
6. return：返回值与函数声明类型一致

# 4. 文件格式与输入子集统一文档

## 4.1 `.l` 文件最小支持子集

### 支持内容
- 定义区中的简单宏
- 规则区
- 用户代码区可先保留原文，不要求完整执行

### 正则最小支持
- 字符字面量
- 字符类 `[a-z]`
- 连接
- 或 `|`
- 闭包 `*`
- 正闭包 `+`
- 可选 `?`
- 括号 `()`

### 动作最小支持
- `return TOKEN_NAME;`
- 跳过空白符
- 记录字面值

## 4.2 `.y` 文件最小支持子集

### 支持内容
- `%token`
- `%left / %right / %nonassoc`
- 产生式规则
- `{ ... }` 语义动作原文保留

### 先不支持或后续支持
- 复杂 C 代码段直接执行
- 高级错误恢复宏
- 极复杂优先级覆盖

# 6. 联调协议

## 6.1 联调顺序

### 第一阶段：A ↔ B
- A 输出固定格式 `TokenStream`
- B 构造一个简单表达式文法，验证 token 能被成功规约

### 第二阶段：B ↔ C
- B 输出 AST
- C 用手工构造或真实 AST 进行符号表构造和表达式类型检查

### 第三阶段：A ↔ B ↔ C
- 从输入源文件直接跑到 IR 输出

## 6.2 联调检查单

### A 交付给 B 前
- [ ] Token kind 已固定命名
- [ ] 关键字 / 标识符冲突规则已统一
- [ ] EOF token 一定输出
- [ ] 行列号有效

### B 交付给 C 前
- [ ] 产生式编号固定
- [ ] AST 节点类型固定
- [ ] 节点 line / column 可追踪
- [ ] 语法错误能定位

### C 集成前
- [ ] 作用域进入退出位置明确
- [ ] 符号表重定义检查完成
- [ ] 基本类型宽度统一
- [ ] IR 输出格式固定
